# MATRIZ 3: ANTIPATRONES ESTOCÁSTICOS (061 - 082)

| ID | Antipatrón | Disfunción Causal | Señal de Presencia | Impacto en Robustez | Refactor (Alternativa) |
|----|------------|-------------------|--------------------|---------------------|------------------------|
| ANT-061 | Test de Aserción Trivíal | Testear que `True == True` o que el constructor de una clase básica no crashea sin testear métodos. | Code Coverage alto (100%) pero los fallos críticos pasan. | Ilusión de Integridad (Coverage Fake) | Mutational Testing y Edge Cases reales. |
| ANT-062 | Inyección de Logs Temporales Eternos | Llenar el código de `print("Aqui llego")` y `logger.debug("Test1")` dejándolos en la rama principal. | SPAM en logs, dificultad para debugear, devaluación del log. | Pérdida Termodinámica Visual | Limpieza de artefactos pre-commit o uso de Tracebacks inteligentes. |
| ANT-063 | Bucle de Retry Ciego en UI | Recargar la página infinita si falla la red en vez de mostrar estado `Offline` al usuario. | Congelación de la interfaz gráfica de CORTEX. | Pésima UX, Calentamiento Local | Componentes de UI Resilientes, Fallback Offline State. |
| ANT-064 | Sobre-Herencia de Clases (Deep Hierarchy) | Un componente hereda 5 niveles de profundidad (Ej. `SuperBaseModelManagerFactory`). | Complejidad de Traceback inmanejable. Imposible leer sin IDE. | Entropía Arquitectónica Incontenible | Preferir Composición sobre Herencia, Duck Typing limpio. |
| ANT-065 | Secuestro del Garbage Collector | Guardar referencias globales en listas estáticas para que Python no borre los objetos de Memoria. | Crecimiento constante de RAM hasta el colapso (OOM). | Fuga de Memoria Severa (Memory Leak) | Uso de `weakref` o limpieza determinista (Delete explicito). |
| ANT-066 | Mantenimiento de Ramas Zombis en Git | Tener 50 ramas locales y remotas `fix/xxx` que llevan 2 años sin borrarse ni cerrarse. | Caos visual, retraso en búsquedas y lentitud del clonado. | Entropía del Repositorio | Auto-Delete de ramas tras Merge/Squash (Git Sentinel). |
| ANT-067 | Inversión Inapropiada de Control (IoC) | Aplicar patrones Java Empresariales (Spring) a scripts de 100 líneas en Python forzando inyecciones complejas. | 90% del código es boilerplate de setup, 10% es valor real. | Anergía Excesiva (Prosa Arquitectónica) | Código Pythonic lineal, importación funcional simple. |
| ANT-068 | Configuración Flotante (Floating Settings) | Variables de configuración definidas aleatoriamente al inicio de 20 archivos distintos (`MAX_RETRIES=5` aquí, `TIMEOUT=10` allá). | Cambiar el comportamiento de un API implica cazar 20 constantes aisladas. | Fallos en cascada por configuración cruzada | Un único archivo `config.py` o Config Object Central Pydantic. |
| ANT-069 | Uso de Shell=True Indiscriminado | Usar `subprocess.run(..., shell=True)` para inyectar strings concatenados (Riesgo). | Vulnerabilidad directa a Command Injection, problemas de escape Bash. | Seguridad Destrozada / Ejecución Indeterminista | Uso estricto de Arrays de argumentos (`shell=False`) pre-parseados. |
| ANT-070 | Parada Estática para Sincronizar | Escribir `time.sleep(5)` esperando que una web o proceso en segundo plano acabe (Mágico). | O falla por lentitud del anfitrión o gasta tiempo inútil si es rápido. | Latencia (Anergía) o Interbloqueos aleatorios | Polling Event-Based o WebHooks / Promesas explícitas (Waits). |
| ANT-071 | Delegar Autenticación a Terceros No C5 | Dejar que NextAuth o servicios Vercel/Clerk manejen el login para un panel de orquestación C5 soberano. | Si Clerk se cae, CORTEX no arranca (Pérdida de la Sovereignity). | Acoplamiento Mortal a Dependencias Cloud | Auth Local SQLite con JWT nativos (Independencia Absoluta). |
| ANT-072 | Excepciones como Control de Iterador | Usar `StopIteration` u otros errores para saltar un if lógico normal. | Ilegibilidad, lentitud en el backend de Python que deoptimiza. | Latencia Computacional | Usar banderas binarias explícitas o `break` normal. |
| ANT-073 | Retornos de Múltiples Tipos (Union Slop) | Función que retorna String si va bien, False si va mal, None si no encontró, y Float si hay cálculo parcial. | Obliga a quien lo invoca a escribir 5 bloques `if isinstance` perdiendo Exergía. | Riesgos Severos de Tipado (TypeErrors ocultos) | Retornar Monadas (Result/Option pattern) o Objetos de Estado Unificados (Pydantic). |
| ANT-074 | Inclusión de Binarios y Pesos Masivos en Git | Comitear un `.pt` de 5GB o directorios `node_modules/` al ledger principal. | Repositorio intransitable, clonarlo tarda horas (Git bloated). | Fricción Topológica Absoluta | Añadir a `.gitignore` y usar LFS o scripts de pre-descarga S3. |
| ANT-075 | Uso de Fechas sin TimeZone en BD | Guardar `datetime.now()` estático en la SQLite. | En despliegues remotos o agentes de distinta zona horaria, los timestamps se solapan o rompen el orden causal (Provenance). | Caos Cronológico Histórico (Inútil para Auditoría) | Almacenamiento Estricto de EPOCH (Entero) o Isoformat UTC Forzoso. |
| ANT-076 | Importación Global Pesada en Módulo Utilitario | Poner `import torch` en un archivo `utils/strings.py` solo porque una función olvidada lo usó. | Cada vez que alguien importa un `util.string`, carga el Kernel de IA en RAM y traba el Mac por 2 segundos. | Asfixia RAM / Cold Start Brutal | Refactorización de imports locales diferidos (`Lazy Import`). |
| ANT-077 | Captura Generalizada y Re-Lanzamiento Vago | Bloque `try...except Exception as e: raise Exception("Fallo algo")`. | Destruye el Traceback Original de C, oscureciendo completamente el contexto de línea donde ocurrió el bug (Traceback Loss). | Blindness de Depuración Total | Rethrow limpio (`raise`) o Wrap conservando el stack trace `raise CortexError from e`. |
| ANT-078 | Optimización Prematura (Premature Optimization) | Perder 5 horas escribiendo una función recursiva memoizada en Cython cuando la tarea solo corre 1 vez al día con N=100. | Pérdida masiva de tiempo, introducción de C-Bugs intratables, código que nadie más entiende. | Anergía Operacional y Deuda Cognitiva | Hacerlo Simple; Hacerlo C5. Solo Optimizar cuando el Profiler de `Health-Cortex` cante sangre (P0). |
| ANT-079 | Encriptado Incompleto de Estado en Tránsito | Mandar JSON crudos inter-agentes en red sin TLS ni hash HMAC local, asumiendo seguridad en Localhost. | Vulnerable a Sniffers locales de otros Daemons en máquinas comprometidas. | Filtración Local de Secretos y API Keys (Vector ADV-007) | Cifrado Mínimo de Túnel o Socket PTY (Singularity Tunneling). |
| ANT-080 | Despreocupación por Path Case Sensitivity | Acceder a `File.txt` en Mac, que funciona local (Case-Insensitive APFS), pero explota estrepitosamente en Linux (Ext4) donde es `file.txt`. | El código muere nada más saltar al Servidor Remoto o Subagente Linux de Sandbox. | Cross-Platform Broken (Bugs Oscuros Inconsistentes) | Estrictez C5: Forzar Lowercase Naming universal en todas las rutas locales y referencias al FS. |
| ANT-081 | Acumulación de Warnings Deprecados Silenciosos | Librerías estándar mandando "DeprecationWarning" continuamente ignorados por meses hasta que actualizan la versión y colapsa (Breaking Change Submarino). | Una actualización de Python 3.12 a 3.14 destruye 10 dependencias simultáneamente de un día para otro sin avisar. | Ruptura Sistémica por Pila Tecnológica (Tech Debt) | Bloquear la versión (Freeze), pero priorizar migración de cada Warning en las sesiones de `Pulir / Health Cortex`. |
| ANT-082 | Confianza Ciega en Output de Subagente No Validado | Parsear el output de texto directo desde `Subagent_A` e inyectarlo en el método `.execute_sql()` del DB. | Injection Attack accidental de código (Drop Tables) si la LLM enloqueció y mandó código letal como parte del string. | Vector Crítico Inyectado / Corrupción de la Persistencia C5 | Usar Parsers intermedios y BFT Consensus (Validadores). Jamás "Execute" directo. |
