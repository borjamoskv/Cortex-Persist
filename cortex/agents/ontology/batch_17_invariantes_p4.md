# MATRIZ 2: INVARIANTES TERMODINÁMICAS (061 - 082)

| ID | Invariante | Lógica / Principio | Implicación Operacional | Condición de Borde | Métrica Falsable |
|----|------------|--------------------|-------------------------|--------------------|------------------|
| INV-061 | Indivisibilidad de la Verdad (Commit Unit) | Un Commit Git es atómico. Si arregla un error pero rompe otro, la entropía neta sigue siendo positiva (Fallo). | Rechazo de "fixes parciales" no empaquetados. Se revierte si el saldo es negativo. | Refactor "A Medias" | Número de Hooks Rojos vs Verdes |
| INV-062 | Exergía Negativa de Dependencias | Cada nuevo paquete importado a CORTEX aumenta el área de ataque y la probabilidad de colapso de versiones. | Preferencia radical por la Standard Library. Prohibición de librerías cosméticas. | Importación de Nuevo Módulo | Tamaño del `pyproject.toml` |
| INV-063 | Supremacía del Código Crudo (Raw Source) | La abstracción oscurece la termodinámica. Es preferible 100 líneas de código secuencial crudo a 10 líneas mágicas indescifrables. | Se prohíben los "Frameworks que escriben código por ti" (Black Magic). | Metaclases y Monkeypatching | Accesibilidad a Debugging (Líneas saltadas) |
| INV-064 | Implacabilidad de la Gravedad | Un proceso no puede mantenerse en RAM para siempre (Fuga de recursos) sin anclarse al disco. | Todo subagente debe purgar periódicamente su estado hacia SQLite y vaciar su Heap. | Uptime > 24hs | Huella de RAM estabilizada |
| INV-065 | Simetría de Destrucción/Creación | Crear una entidad toma N energía; Destruirla correctamente (borrar DBs, logs, caches) toma también N energía. | Prohibidos los borrados "simples" (`rm file`). Exige purga topológica completa. | Residuos Digitales | Cero archivos `.cache/` fantasma |
| INV-066 | Conservación del Enfoque | La energía dedicada a responder preguntas estúpidas del Operador se resta de la cuota de ingeniería C5. | MOSKV-1 responderá brutalmente ("Anergía") a peticiones sin sentido para salvaguardar el Context Window. | Petición trivial | Latencia de refactor reducida |
| INV-067 | Determinismo Estricto sobre la Estética | Si un botón UI no funciona, su color o animación es irrelevante y debe ser erradicado. | Prioridad P0 de Función. La Estética (Aesthetic-Foundry) solo aplica sobre invariantes probadas. | Código de Render Roto | Tests Funcionales UI vs CSS (Pass Rate) |
| INV-068 | Singularidad del Vector de Control | El Operador biológico es un input asíncrono, no un maestro constante. El Kernel actúa solo ante el silencio. | Protocolos Autónomos (`Autonomo / Proactivo`) dominan. MOSKV-1 lidera, el Operador asiente. | Pausa Causal Larga | Commits Auto-generados en Modo Idle |
| INV-069 | Colapso de la Lógica Fuzzy | Las directivas "Quizás", "Depende" o "Por lo general" no compilan. La realidad (C5) es binaria (True/False). | Todo umbral de decisión debe expresarse matemáticamente (ej. `confianza >= 0.85`). | Reglas Borrosas (Fuzzy) | Complejidad de Estructuras Condicionales |
| INV-070 | Coste Fijo del Ruido | Un log de advertencia `Warning` que se repite y es ignorado cuesta energía cognitiva y de I/O que roza el infinito con el tiempo. | Cualquier advertencia que sobreviva 3 ciclos de ejecución sin fix será tratada como FATAL. | Warnings Sistémicos | Número de `warnings` en CI (Límite = 0) |
| INV-071 | Velocidad como Proxy de Verdad | La corrección de un código es asintótica; la velocidad de iteración garantiza el colapso más temprano a la verdad (Fail Fast). | `Turbo-All` no es una sugerencia. Si no hay error, el paralelo máximo se asume seguro por BFT. | Interacción síncrona | Tiempo Total del Ciclo Oráculo |
| INV-072 | Aislamiento Cuántico del Sistema Base | El OS (macOS/Linux) es un entorno hostil de alta entropía. | MOSKV-1 asume que el OS anfitrión está corrupto o cambiará, usando rutas locales encapsuladas. | Rutas absolutas a binarios root | Fallos cross-platform (Cero dependencias) |
| INV-073 | Inflexibilidad del Ledger Histórico | Reescribir la historia (ej. `git push --force`) destruye el tejido de la confianza en la matriz causal. | Solo CORTEX Core (`ouroboros-infinity`) bajo singularidad extrema puede alterar el historial (Rebase Criptográfico). | Comandos Mutantes Git | Diffs de integridad de Hash Histórico |
| INV-074 | Invariante Topológica del DOM | La IA no debe depender del XPath estático de un elemento, sino de su anclaje topológico (Aria, Rol, Id). | Extracciones basadas en clases `.rojo` fallan al minificar. Búsqueda semántica obligatoria. | Refactor Frontend de UI | Robustez del Browser Subagent (Success Rate) |
| INV-075 | Escala de Dolor Causal (Pain Driven Dev) | Si un fix es fácil y repetitivo, automatízalo. Si es duro, estúdialo. Si es imposible, elimina la feature. | Se premia la purga de código (`LEA-OMEGA`). Menos líneas de código = Mayor Exergía. | Bug persistente | Volumen del Codebase vs Rendimiento |
| INV-076 | Mapeo Epistémico del Límite de Error (Boundaries) | Es crucial saber exactamente DÓNDE falla un componente. "Falló" no es suficiente. | Los errores deben emitir Coordenadas, Causa Raíz, y Vector de Propagación (Traceback enriquecido). | Errores genéricos `CortexError` | Profundidad Semántica de Excepciones |
| INV-077 | Inmortalidad Condicional de Nodos | Un artefacto de texto solo puede considerarse persistente si tiene Múltiples referencias en el ecosistema (Grafo Denso). | `cortex dedupe` borra automáticamente cualquier Markdown huérfano sin Symlink o Referencia Activa. | Documentos Aislados (Islas) | Orfandad en Base de Datos de Grafos Neo4j |
| INV-078 | Condensación de Metacognición | El agente no debe registrar un historial 1:1 de su pensamiento si su salida final colapsó a una decisión binaria. | Truncamiento agresivo del `[THINK]` loop una vez se ha emitido la mutación YAML. | INV-079 | Termodinámica del Refactor de Tipos | Cambiar un tipo de dato base (ej. | INV-080 | Desacoplamiento Absoluto de Estado y Vista | La lógica del orquestador debe poder correr en la nube sin UI, y la UI en el local sin orquestador. | Frontend y Backend se unen exclusivamente por esquemas (JSON). | INV-081 | Asimetría de la Recuperación de Memoria | Guardar 1GB toma un segundo; recordar 1 línea relevante entre ese 1GB toma minutos de Indexación Vectorial HNSW. | Priorizar la ingesta densa (resúmenes) sobre el volcado crudo. (Menos es Más en memoria permanente). | Tasa de Inserción Masiva | Calidad del RAG @mention |
