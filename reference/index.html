    Sovereign Gate¶

    Rate limiting, request approval workflows, and action-level security enforcement.

    cortex.extensions.gate.core.SovereignGate ¶

    L3 Action Interception Middleware.

    Every action classified as L3_EXECUTE or L4_MUTATE must pass through this gate. The gate generates an HMAC-SHA256 challenge that the operator must sign before execution proceeds.

    Modes
    • ENFORCE: Block execution until operator signs (production)
    • AUDIT_ONLY: Log the action but allow execution (development)
    • DISABLED: Transparent passthrough (testing)

    Methods:¶

    approve(action_id: str, signature: str, operator_id: str = 'operator') -> bool ¶

    Approve an action with HMAC signature verification.

    The signature must match the challenge generated during request_approval.

    approve_interactive(action_id: str) -> bool ¶

    Interactive CLI approval - prompts the operator directly.

    In AUDIT_ONLY mode, auto-approves with a log entry. In DISABLED mode, does nothing.

    deny(action_id: str, reason: str = '') -> None ¶

    Explicitly deny a pending action.

    execute_subprocess(action_id: str, cmd: list[str], **kwargs: Any) -> subprocess.CompletedProcess ¶

    Execute a subprocess ONLY if the action is approved.

    This is the single choke-point for all L3 subprocess calls.

    get_audit_log(limit: int = 50) -> list[dict[str, Any]] ¶

    Return the most recent audit log entries.

    get_pending() -> list[PendingAction] ¶

    Return all pending actions, expiring stale ones first.

    get_status() -> dict[str, Any] ¶
