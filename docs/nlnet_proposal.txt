CORTEX-Persist is an open-source cognitive governance layer for autonomous AI agents. It provides cryptographic audit trails, epistemic containment, and formal trust boundaries for AI systems that use external tools (via the Model Context Protocol, MCP).

The problem: As AI agents (Claude, GPT, Gemini) gain the ability to autonomously read files, execute code, browse the web, and modify databases via MCP, there is no standardized mechanism for:

Non-repudiable attribution: knowing which agent did what, when, and why
Tamper-evident decision logging: ensuring that AI-generated decisions cannot be silently altered after the fact
Epistemic containment: preventing generative hallucinations from contaminating persistent state
What CORTEX does: Every tool call an AI agent makes is shadowed by an immutable, SHA-256 hash-chained ledger entry. This is not authorization — it is forensic auditability. CORTEX makes the cost of unattributed AI action structurally infinite.

Expected outcomes:

A production-ready MCP compliance sidecar that any AI agent can adopt
A formal trust boundary specification for stochastic-to-deterministic state transitions
EU AI Act Article 12 (record-keeping for high-risk AI) reference implementation
Published research on cryptographic governance of non-deterministic AI systems
Stack: Python 3.10+, SQLite + sqlite-vec, ONNX Runtime, FastAPI, SHA-256 hash chains. All code Apache-2.0 on GitHub.

5. Significant Technical Challenges
The Genesis Problem: Hash chains must be anchored to an externally verifiable initial state. Without a trusted genesis, the entire chain is self-referential. CORTEX solves this via environment-configurable anchors (CORTEX_LEDGER_ANCHOR), but a publishable specification for multi-agent genesis consensus is needed.

Stochastic-to-Deterministic Boundary: LLM outputs are probabilistic. Defining the precise interface where generative conjecture crosses into deterministic truth (the "guard boundary") requires formal specification, not just code.

Performance at Scale: SHA-256 hash chains add ~18ms per write. For high-throughput agent swarms (100+ concurrent agents), batch verification and Merkle tree summarization are needed without breaking chain continuity.

Cross-Agent Provenance: When Agent A's output becomes Agent B's input, causal taint must propagate through the DAG. This requires a novel taint-tracking protocol that doesn't exist in current MCP specifications.
