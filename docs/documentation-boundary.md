# Documentation Boundary

This `docs/` directory is the tracked source for the public documentation surface and also preserves stable repo-root entrypoints for technical review, procurement workflows, and due-diligence checklists.

If you are evaluating CORTEX for the first time, start with [Supported Core](supported-core.md) and [Canonical Demo](canonical-demo.md). The published docs may still be deployed through a separate build surface, but the source of truth for the homepage and the core reference pages listed in the next section lives here in-tree. The buyer-facing companion documents remain intentionally anchored at the repository root.
