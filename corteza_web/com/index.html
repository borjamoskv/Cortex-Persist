How It's Verified

With reproducible tests, not promises or pretty dashboards.

01

Capture

Every relevant event generates automatic logs. Normal input, errors, timeouts, retries, model changes, human intervention, policy blocks.

capture_rate ≥ 99.8% 0 orphan events · unique trace_id per event
02

Integrity

Logs aren't retroactive play-dough. Each event hash chains to the previous one. Any later alteration breaks the chain and is detected.

tamper_detection = 100% hash_n = H(event_n + hash_(n-1))
03

Enforcement

CORTEX doesn't just watch. It blocks or flags: inference without log, unapproved model, silent prompt change, critical output without review, deletion outside retention.

Each policy: condition + response + evidence Reproducible. If not, it's decoration.
04

Audit

"Show me Tuesday 14:00–15:00 inferences from system X with model Y." Events, versions, operator, signed export. In minutes, not days.

search <30s · export <2min · integrity <5min If it takes two days, it's compliance cosplay.

FAQ

Does CORTEX guarantee no fines?

No. It reduces regulatory exposure and strengthens your technical and evidentiary position.

Does it work if we're already in production?

Yes. It generates evidence from activation and can reconstruct partial history if prior sources exist.

Does it replace the legal team?

No. It replaces the lack of technical evidence.

Is it only for high-risk systems?

No. It also serves internal governance, incident response, vendor oversight, and contractual traceability.

Does Art. 12 require cryptographic hashing?

Not literally. It requires technical capability for automatic logging. The cryptographic architecture is our design choice to ensure evidence survives real scrutiny.

Does it integrate with any model?

Yes. CORTEX is model-agnostic. Multi-model neutrality is a core design principle, not an afterthought.
