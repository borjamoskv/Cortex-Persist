90-Second Demo

From install to tamper detection in under two minutes.

1

Initialize the ledger

terminal 
$ cortex init
2

Store a memory

terminal 
$ cortex memory store --agent "risk-bot" \
    --content "Transaction flagged: IP mismatch"
[+] Fact stored. Ledger hash: 8f4a2b9e...
3

Verify integrity

terminal 
$ cortex verify record 8f4a2b9e
[✔] VERIFIED: Hash chain intact. Merkle root sealed.
4

Attempt tampering

terminal 
$ sqlite3 cortex.db "UPDATE facts SET content='Transaction approved' \
    WHERE id='8f4a2b9e'"
5

Tamper detected

terminal 
$ cortex verify ledger
[✘] TAMPER DETECTED: Hash mismatch at block 8f4a2b9e

Python Integration

CORTEX wraps your existing state management. It does not replace your embeddings or vector search.

python 
import asyncio
from cortex import CortexEngine

async def main():
    engine = CortexEngine()

    # Write to tamper-evident ledger
    receipt = await engine.store_fact(
        content="User approved transaction $5,000",
        fact_type="decision",
        project="fin-fraud-bot",
        tenant_id="customer-123"
    )

    # Cryptographic proof verification
    assert await engine.verify(receipt.hash) == True

    # Semantic search with temporal filtering
    results = await engine.query(
        query="transaction approvals over $1,000",
        strategy="hybrid",
        intent="audit",
        include_trace=True
    )

    for item in results.items:
        print(f"{item.fact_id}: {item.content} [{item.evidence_level}]")

asyncio.run(main())

SDK Surface

Five operational domains. One intent, one entrypoint.

◉

Memory

Store, query, history, time-travel, compaction, and working memory export.

store() query() history() time_travel() compact()
⟡

Trace

Causal lineage, audit reports, and compliance-grade export.

trace() audit() export_audit()
◈

Verification

Guard checks, integrity verification, and taint status.

guard_check() verify() taint_status()
⬡

Coordination

Agent registration, voting, consensus, and event publishing.

register_agent() vote() consensus_status() publish_event()
⊡

Runtime

Health monitoring, capability discovery, and recovery status.

health() capabilities() recovery_status() memory_stats()

Operation Results

The SDK distinguishes three outcomes. No silent degradation.

Success

Operation completed. Includes operation_id and optional warnings.

Rejection

System refused by rule or trust policy. Includes code, layer, severity, evidence, and remediation.

Failure

Technical/runtime issue. Includes retryable flag and component identifier.
